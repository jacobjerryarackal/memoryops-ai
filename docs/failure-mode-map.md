# MemoryOps AI — Failure Mode Mapping

This document maps key operational failure modes relevant to the MemoryOps AI read and write paths, along with their corresponding invariants, mitigations, test validations, and residual risks.

| Failure Mode | Affected Concern | Local Mitigation / Invariant | Existing Test Evidence | Residual Risk |
| :--- | :--- | :--- | :--- | :--- |
| **Cross-Tenant Memory Leakage** | `Phase 11 (Security)` | Repository query filters by `tenant_id`. Retriever defensively drops non-matching tenant records. | `tests/test_repository.py`, [test_retrieval_telemetry.py](file:///d:/AI/memoryops-ai/tests/test_retrieval_telemetry.py#L425) | Multi-tenant database connection failures could bypass logic if connection pool isolation fails. |
| **Cross-User Memory Leakage** | `Phase 11 (Security)` | Repository query filters by `user_id`. Retriever defensively drops non-matching user records. | `tests/test_repository.py`, [test_retrieval_telemetry.py](file:///d:/AI/memoryops-ai/tests/test_retrieval_telemetry.py#L425) | User session pollution at the routing/session management layer. |
| **Soft-deleted Memory Recall** | `Phase 06 (Memory)` / `Phase 11 (Security)` | Repository query excludes `deleted` status. Retriever defensively skips non-active records. | `tests/test_repository.py`, [test_retrieval_telemetry.py](file:///d:/AI/memoryops-ai/tests/test_retrieval_telemetry.py#L425) | None. Prevented at both repository and service boundaries. |
| **Archived / Pending / Rejected Memory Leak** | `Phase 06 (Memory)` / `Phase 11 (Security)` | Repository query filters to `active` only. Retriever defensively skips non-active records. | `tests/test_repository.py`, [test_retrieval_telemetry.py](file:///d:/AI/memoryops-ai/tests/test_retrieval_telemetry.py#L425) | None. Enforced at both query and retrieval service layers. |
| **Stale Embeddings After Mutation** | `Phase 06 (Memory)` | Writing content updates clears the stored `embedding` to `None`. Semantic query excludes `embedding == None`. | `tests/test_repository.py` (`test_repo_update_content_invalidates_embedding`) | None. Verified by write services. |
| **Incompatible Embedding Model Spaces** | `Phase 06 (Memory)` | Dimension validation requires exactly 1536 floats. | `tests/test_openai_embedding.py` (dimension checking only) | **High**. Callers can write arbitrary 1536-dimensional vectors from different models. Model provenance is not runtime-enforced. |
| **Embedding Provider Failure** | `Phase 05 (LLM)` / `Phase 12 (Reliability)` | Coordinator catches provider exceptions and degrades to fallback lexical search. | `tests/test_retrieval_services.py` (`test_coordinator_fallback_on_embedding_exception`) | Temporary loss of semantic relevance capabilities. |
| **Malformed Embedding Provider Response** | `Phase 05 (LLM)` / `Phase 12 (Reliability)` | `OpenAIEmbeddingService` validates dimensions, finite floats, and non-NaN values, throwing on anomalies. | `tests/test_openai_embedding.py` | New API output shapes could pass validation but cause downstream failures (mitigated by broad try/except fallback). |
| **Arbitrary Semantic Candidate Bounding** | `Phase 06 (Memory)` | Repository performs similarity queries over the full scoped active pool (no pre-filtering first-N). | *None* (Verified via code inspection of `InMemoryMemoryRepository.search_candidates`) | High memory usage when the candidate pool scales very large (deferred to pgvector index tuning). |
| **Non-Deterministic Ranking Ties** | `Phase 06 (Memory)` | Timsort sequentially sorts by `final_score` DESC, `created_at` DESC, then UUID `id` ASC. | `tests/test_retrieval_services.py` (tie-breaking tests) | None. Sorting is mathematically deterministic. |
| **Oversized Context Injection** | `Phase 06 (Memory)` / `Phase 12 (Reliability)` | Context Composer enforces character and count budgets, skipping oversized records. | `tests/test_retrieval_services.py` (composer budgets) | Character count does not perfectly reflect LLM tokenizer token count (potential minor budget slippage). |
| **Temporary Chat Memory Retrieval** | `Phase 04 (Orchestration)` / `Phase 06 (Memory)` | Router and coordinator bypass retrieval and embedding steps when `temporary_chat == True`. | `tests/test_retrieval_telemetry.py` (`test_temporary_chat_telemetry_and_no_invocation`) | None. Bypassed successfully. |
| **Telemetry Failure Blocks Reads** | `Phase 10 (Observability)` / `Phase 12 (Reliability)` | `StructuredRetrievalLogger` catches logging errors. Coordinator wraps emit call in broad try/except. | `tests/test_retrieval_telemetry.py` (`test_telemetry_emission_failure_isolation`) | Under extreme disk/logging volume stress, retrieval logs could fail to emit. |
| **Governance Audit Log Pollution** | `Phase 10 (Observability)` / `Phase 15 (Governance)` | Telemetry writes to operational process logs. Governance audit records are handled separately. | `tests/test_retrieval_telemetry.py` (`test_audit_separation`) | None. Separation is structurally enforced. |
