# MemoryOps AI — Failure Mode Mapping

This document maps key operational failure modes relevant to the MemoryOps AI read and write paths, along with their corresponding invariants, mitigations, and test validations.

| Failure Mode | MemoryOps Invariant or Mitigation | Authoritative Document | Existing Test Evidence | Status |
| :--- | :--- | :--- | :--- | :--- |
| **Cross-Tenant Memory Leakage** | Repository query strictly filters `tenant_id == tenant_id` and `user_id == user_id`. Retriever performs defensive secondary validation. | `docs/design/retrieval-spine.md`, `ADR-005` | `tests/test_repository.py`, `tests/test_retrieval_telemetry.py:test_retriever_defensive_boundary_scope_and_active` | **MITIGATED & TESTED** |
| **Deleted Memory Recall** | Soft-deleted memories (`status == 'deleted'`) are excluded at the repository layer. Retriever defensively skips non-active records. | `infra/adr/ADR-005-deletion-guarantee.md` | `tests/test_repository.py`, `tests/test_retrieval_telemetry.py:test_retriever_defensive_boundary_scope_and_active` | **MITIGATED & TESTED** |
| **Stale Embeddings After Mutation** | Content update or semantic mutation clears the stored `embedding` to `None`. Semantic query excludes `embedding == None` records. | `infra/adr/ADR-006-memory-identity-and-write-path-mutation.md` | `tests/test_repository.py:test_repo_update_content_invalidates_embedding` | **MITIGATED & TESTED** |
| **Cross-Model Embedding Comparison** | Input vectors must be exactly 1536 dimensions. Provenance validation is deferred to future write path updates. | `infra/adr/ADR-007-embedding-provider-and-model.md` | `tests/test_openai_embedding.py` (dimension checking only) | **PARTIALLY MITIGATED** (Provenance not runtime-enforced) |
| **Semantic Provider Failure** | Embedding generation exceptions are caught by `RetrievalCoordinator`, failing back gracefully to active lexical-only search. | `docs/design/retrieval-spine.md` | `tests/test_retrieval_services.py:test_coordinator_fallback_on_embedding_exception`, `tests/test_retrieval_telemetry.py` | **MITIGATED & TESTED** |
| **Arbitrary Candidate Pre-filtering** | No first-N semantic pre-filter is executed before similarity evaluation. The full scoped candidate pool is evaluated. | `docs/design/retrieval-spine.md` (Section 8) | Verified via repository queries. | **MITIGATED** |
| **Non-Deterministic Ranking Ties** | Ties are broken deterministically in order: `final_score` DESC, `created_at` DESC, `id` ASC (lexicographical string UUID sort). | `docs/design/retrieval-spine.md` (Section 7) | `tests/test_retrieval_services.py` (tie-breaking tests) | **MITIGATED & TESTED** |
| **Context Budget Overflow** | Composer skips oversized memories that would exceed character bounds and checks subsequent memories. Content is never truncated. | `docs/design/retrieval-spine.md` (Section 9) | `tests/test_retrieval_services.py` (budget skip tests) | **MITIGATED & TESTED** |
| **Telemetry Failure Blocks Reads** | Telemetry sink catches serialization/logging errors internally. Coordinator wraps telemetry invocation to prevent propagation. | `infra/adr/ADR-004-observability.md` | `tests/test_retrieval_telemetry.py:test_telemetry_emission_failure_isolation` | **MITIGATED & TESTED** |
| **Speculative Implementation** | Agent proceeds to write code when a dependency is absent or undefined. | `AGENTS.md` (Before Starting Any Task) | Checked at design gate. | **MITIGATED via PHASE GATE** |
